# 翻譯語氣與風格指南

## 核心原則
1. **專業金融口吻**：語氣應像金融機構研究報告，嚴謹且精確。
2. **口語化專業感**：雖然是財報文字，但在翻譯對話（問答環節）時，應保留「人與人對話」的自然感，但用詞需專業。
3. **台灣在地化**：嚴格使用台灣用語（例如「資訊」而非「信息」、「軟體」而非「軟件」）。
4. **數字與日期格式**：
   - 保持原始數字精確度。
   - 日期格式如 2026-05-05 或 2026年5月5日皆可。
## 結構對照
1. **段落結構**：採用「英文段落 \n 中文譯文」的對應結構。段落之間留一個空行。
2. **發言者標註**：
   - **發言人姓名**（如 Lisa Su）：使用 **粗體** 並加上角色標籤。
   - **問答環節**：問題部分 (Q) 使用 `> ` 引用塊格式，答案部分 (A) 維持正常。
3. **數據完整性**：所有數字、百分比、日期必須 100% 保持原始精確度，嚴禁自行進行四捨五入。
4. **表格處理**：若 PDF 包含財務表格，請使用 Markdown Table 重新整理，確保數據易於閱讀。

## 範例 (分析師友善格式)

**Lisa Su (CEO)**:
"We are seeing strong momentum as customers move from pilots to large-scale production deployments."
「隨著客戶從試點轉向大規模生產部署，我們看到了強勁的動能。」

> **Q - Analyst (Bank of America)**:
> "How should we think about the GPU supply for the rest of the year?"
> 「我們應該如何看待今年餘下時間的 GPU 供應情況？」
